Introduction

Text Automata allows you to quickly generate text from a template. You can create the markdown for a template using the tools on the template page and generate the text from the template on the generate page (by clicking the "next" button on the template page). The tool is useful for dynamically generating text for repetitive tasks such as cover letters and applications.

Toggles

Toggled text is text that you can include (toggle on) or exclude (toggle off). It is represented in the markdown using square brackets [ ]. Here is an example of a template with two toggles:

Hi, [we are excited to offer you a position at Text Automata!][unfortunately, we will not be able to extend an offer at this time.]

There are two ways to create a toggle on the template page:

1. Select/highlight the text you want to toggle in the main input section and click the toggle button. NOTE: If you do not select or highlight any text before clicking the toggle button, the square brackets [ ] will not appear.
2. Type the '[' and ']' characters around the text you want to toggle.

By default, all toggled segments are toggled on for visibility.

Hi, we are excited to offer you a position at Text Automata!unfortunately, we will not be able to extend an offer at this time.

If you click the text, the toggle will be crossed out:

Hi, we are excited to offer you a position at Text Automata!unfortunately, we will not be able to extend an offer at this time.

You can hide all excluded (crossed-out) by changing the page settings in the menu bar.

Hi, unfortunately, we will not be able to extend an offer at this time.

Variables

You can type anything into variable text. It is represented in the markdown using braces { }.

I would like to apply for the {role} at {company}.

There are three ways to create a variable on the template page:

1. Select/highlight the text you want to make into a variable and click the variable button.
2. Type the '{' and '}' characters around the text you want to make into a custom prompt.
3. Click the variable button, enter the variable name, and click finish. NOTE: There is a character limit on the variable name length with the template user interface. If you want to use a longer variable name, you must manually select and/or enter the '{' and '}' characters around the variable name.

Here is what you will see first for the template from the previous example:

I would like to apply for the role at company.

By default, the prompt text is shown. Click on the variable to edit it. Variables that you are currently editing will be highlighted.

I would like to apply for the role at company.

By default, the prompt text is shown. You can type anything to overwrite the prompt. You can create a new line by hitting enter. Hit the Esc key to finish editing. If you do not hit the Esc key after editing, you will not be able to edit other parts of the text. To copy and paste text into a variable, use the legacy generator linked at the bottom of the template page.

I would like to apply for the frontend developer role at company.

Your final text after editing both variables will look like this:

I would like to apply for the frontend developer role at Text Automata.

Here is another sentence you can generate with the same template:

I would like to apply for the keyboard designer position at Artemis Design Studios.

The autoapply setting assigns the same value to all variables with the same name.

I was excited when I saw the opportunity at {company}. I believe that my experience in {experience} will allow me to thrive at {company}.

When you select company, all variables with the name company will be highlighted.

I was excited when I saw the opportunity at company. I believe that my experience in experience will allow me to thrive at company.

You can now edit both variables simultaneously:

I was excited when I saw the opportunity at Text Automata. I believe that my experience in experience will allow me to thrive at Text Automata.

You can turn off autoapply by clicking the autoapply button. With autoapply turned off, you can overwrite the text for the second company entry.

I was excited when I saw the opportunity at Text Automata. I believe that my experience in experience will allow me to thrive at Artemis Design Studios.

Nesting

You can nest variables within toggles. Here is an example of valid nesting:

[As an avid follower of the {industry}, I was excited about the opportunity at {company}.] [While I am unfamiliar with the {industry} I believe my experience in {experience} will allow me to thrive at {company}.]

If you exclude and hide a nested toggle, the variables within the nested toggle will be hidden.

Escape Character

You can use the escape character '\' (backslash) to enter text that would otherwise be evaluated. Here are some examples in a template:

1. arr[0]
2. arr\[0]
3. Here is a \\

Here is what the template evaluates to:

1. arr0
2. arr[0]
3. Here is a \

Legacy Generator

The original Text Automata generator provided a step-by-step interface to edit each segment individually. If you are concerned about forgetting variables or copying and pasting text, you may find the legacy generator more useful. The legacy generator supports hidden toggles and autoapply (in the form of autofill) as well. You can use the legacy generator by selecting the legacy button on the template page.

Preview

The preview button on the template page allows you to preview how your text will be semantically parsed. It displays the text without the parsed braces or brackets while including colors (like the examples on this page) to display a preview of the parsing.

Import and Download

You can import an existing template into the template page. The preferred file format is .txt for imports but it works with other formats as well. You can download the template and the generated text as .txt files.
