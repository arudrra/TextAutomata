text automata

Introduction

Text Automata allows you to quickly generate text from a template. You can create the markdown for a template using the tools on the template page and generate the text from the template on the generate page (which can be reached by clicking the "next" button on the template page). The tool is useful for dynamically generating text for repetitive tasks such as cover letters and applications.

Toggle

Toggled text is text that you can include (toggle on) or exclude (toggle off). It is represented in the markdown using square brackets [ ]. Here is an example of a template with two toggles:

Hi, [that time works well!][unfortunately, I won't be able to make it.]

There are two ways to create a toggle on the template page:

1. Select/highlight the text you want to toggle in the main input section and click the toggle button. NOTE: If you do not select or highlight any text before clicking the toggle button, the square brackets [ ] will not appear.
2. Type the '[' and ']' characters around the text you want to toggle.

While stepping through the template on the generate page, you will be prompted to include or exclude each toggled segment. By default, a toggled segment is included (toggled on) for visibility. Here is what you will see first for the template from the previous example:

Hi, that time works well!

If you click the exclude button, the toggle will be crossed out:

Hi, that time works well!

After you click the "next" button, the excluded toggle will disappear since it is not included in the final text and the next toggle will appear:

Hi, unfortunately, I won't be able to make it.

Variable

You can type anything into variable text. It is represented in the markdown using braces { }.

I would like to apply for the {role} at {company}.

There are three ways to create a variable on the template page:

1. Select/highlight the text you want to make into a variable and click the variable button.
2. Type the '{' and '}' characters around the text you want to make into a custom prompt.
3. Click the variable button, enter the variable name, and click finish. NOTE: There is a character limit on the variable name length with the template user interface. If you want to use a longer variable name, you must manually enter the '{' and '}' characters around the variable name.

The generate page will sequentially prompt you to fill out the variables. Here is what you will see first for the template from the previous example:

I would like to apply for the role.

By default, the prompt text is shown. You can type anything into the input box to overwrite the prompt. After entering "frontend developer role" and selecting next, you will be prompted for the next variable.

I would like to apply for the frontend developer role at company.

After entering "Text Automata" for the company, the final text will look like this:

I would like to apply for the frontend developer role at Text Automata.

Here is another example of a sentence you can generate with the same template:

I would like to apply for the keyboard designer position at Artemis Design Studios.

If you have autofill turned on, the previous variable entry for the same prompt will be displayed instead of the prompt. You can still change the variable entry by typing anything into the text input box. Here is a template to demonstrate how autofill works:

I was excited when I saw the opportunity at {company}. I believe that my experience in {experience} will allow me to thrive at {company}.

You will be prompted for the company initially since there was no previous entry for company.

I was excited when I saw the opportunity at company

After filling out the company entry as Text Automata and the experience entry as HTML, CSS, and Javascript, the second company prompt will be autofilled with Text Automata.

I was excited when I saw the opportunity at Text Automata. I believe that my experience in HTML, CSS, and Javascript will allow me to thrive at Text Automata

You can still enter any text for the second company entry and override the Text Automata suggestion. You can turn autofill on or off by clicking the autofill button.

Nesting

You can nest variables within toggles. Here is an example of valid nesting:

[As an avid follower of the {industry}, I was excited about the opportunity at {company}.] [While I am unfamiliar with the {industry} I believe my experience in {experience} will allow me to thrive at {company}.]

The toggle is evaluated before the nested variables. If you choose to exclude a nested toggle, you will not be prompted for the nested variables inside the toggle. Instead, you will be prompted to make the next decision.

Escape Character

You can use the escape character '\' (backslash) to enter text that would otherwise be evaluated. Here are some examples in a template:

1. arr[0]
2. arr\[0]
3. Here is a \\

Here is what the template evaluates to:

1. arr0
2. arr[0]
3. Here is a \

Preview

The preview button on the template page allows you to preview how your text will be semantically parsed. It displays the text without the parced braces or brackets while including colors (like the examples on this page) to display a preview of the parsing.

Import and Download

You can import an existing template into the template page. The preferred file format is .txt for imports but it works on other formats as well. You can download the template and the generated text as .txt files.
