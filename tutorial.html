text automata

Introduction

Text Automata allows users to quickly generate text from a template. Once a template is created, the user can iterate through three different semantic choices as defined in the template.

Toggle

Toggled text is text that you can include (toggle on) or exclude (toggle of). It is represented in the markdown using square brackets [ ]. Here is an example toggle template:

Hi, [that time works well!][unfortunately, I won't be able to make it.]

While stepping through the template on the generate page, the user will be prompted to include or exclude each toggled segment. By default, a toggled segment is included (toggled on) for visibility. Here is an example of a user stepping through the example toggle template:

Hi, that time works well!

The user selects the exclude button and the toggle is crossed out:

Hi, that time works well!

The user continues to the next segment. The excluded toggle is hidden for brevity and the next toggle is displayed:

Hi, unfortunately, I won't be able to make it.

Variable

Variable text allows the user to dynamically change a segment of text while generating the final text. It is represented in the markdown using braces { }.

I would like to apply for the {role} at {company}.

The generate page will prompt the user to fill out the role and company variables. Here are two different sentences that can be generated from the same template:

I would like to apply for the keyboard designer position at Artemis Design Studios.

I would like to apply for the frontend developer role at Text Automata.

Nesting

The user can nest variables within toggles. Here is an example of valid nesting:

[As an avid follower of the {industry}, I was excited about the opportunity at company.] [While I am unfamiliar with the {industry} I believe my experience in experience will allow me to thrive at company.]

Escape Character

The escape character '\' allows you to escape text that would otherwise be evaluated.

arr\[0] -> arr[0]
