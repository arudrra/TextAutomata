Introduction

Text Automata was initially developed as a command line tool to generate cover letters for job applications. You can use it to dynamically generate text with highly repetitive content such as cover letters, large volume emails, college admissions essays, and more.

How It Works

The first step in the process is creating a (.txt) source template. You can create the source template using the tools on the template page. For example, consider the following source template for a cover letter:

To whom it may concern,

I would like to apply for the {position} position at {company}. [As an avid follower of the {industry}, I was excited to hear about the opportunity at {company}.] Having read through the qualifications, I believe that my experience in {experience} will allow me to contribute and thrive in the {environment} environment fostered by {company}.

[I developed Text Automata, a tool designed to help people generate text with repetitive parts such as Cover Letters. I used HTML, CSS, and Javascript to develop the website and hosted it on AWS. ][My friend and I designed and developed the Paragon, a 75% mechanical keyboard that was sold worldwide. I used Fusion360 to design the keyboard and created manufacturing drawings for factories.]

I would be delighted to further elaborate upon my experiences in an interview. My preferred mode of communication is via email at {email} or by phone at {phone number}. Thank you for your consideration,

Sincerely,

{name}

You can view and edit this source template for the sample cover letter here.

Text Automata parses the source template and provides you with an interface to dynamically change parts of the text. Here are a few different cover letters that can be generated from the preceding sample template:

To whom it may concern,

I would like to apply for the software engineering position at Text Automata. Having read through the qualifications, I believe that my experience in HTML, CSS, Javascript, and AWS will allow me to contribute and thrive in the data-driven environment fostered by Text Automata.

I would be delighted to further elaborate upon my experiences in an interview. My preferred mode of communication is via email at arudrra@r00dy.com or by phone at 1-(111)-111-1111. Thank you for your consideration,

Sincerely,

Arudrra

To whom it may concern,

I would like to apply for the keyboard designer position at Artemis Design Studios. As an avid follower of the mechanical keyboard space, I was excited to hear about the opportunity at Artemis Design Studios. Having read through the qualifications, I believe that my experience in CAD and manufacturing will allow me to contribute and thrive in the innovative environment fostered by Artemis Design Studios.

My friend and I designed and developed the Paragon, a 75% mechanical keyboard that was sold worldwide. I used Fusion360 to design the keyboard and created manufacturing drawings for factories.

I would be delighted to further elaborate upon my experiences in an interview. My preferred mode of communication is via email at r00dy@keyboarddesigner.com or by phone at 1-(999)-999-9999. Thank you for your consideration,

Sincerely,

Roody

Toggles

Toggled text is text that you can include (toggle on) or exclude (toggle off). You can define toggles in the source template using square brackets [ ].

The following is an example of a source template with two toggles:

Hi, [we are excited to offer you a position at Text Automata!][unfortunately, we will not be able to extend an offer at this time.]

There are two ways to create a toggle on the source template page:

1. Select/highlight the text you want to toggle in the main input section and click the toggle button. NOTE: If you do not select or highlight any text before clicking the toggle button, the square brackets [ ] won't display.
2. Type the '[' and ']' characters around the text you want to toggle.

When the toggles setting is set to show (default), Text Automata displays all toggled text in the generated template whether or not the text is excluded. When the toggles setting is set to hide, Text Automata displays only toggled text selected for inclusion in the generated template.

For example, the following template for the preceding example toggles displays both toggles because toggles setting is set to show:

Hi, we are excited to offer you a position at Text Automata!unfortunately, we will not be able to extend an offer at this time.

You can exclude a toggle by clicking on the text. Text Automata displays a strikethrough excluded toggles:

Hi, we are excited to offer you a position at Text Automata!unfortunately, we will not be able to extend an offer at this time.

You can hide all excluded toggles by clicking the toggles setting in the menubar. For example, the following shows the included toggle and hides the excluded toggle in the generated template:

Hi, unfortunately, we will not be able to extend an offer at this time.

Variables

Variable text is text that you can dynamically replace in your generated template. If you define multiple instances of a variable in your template, you can replace all instances of the variable or selected instances of a variable in your generated template. You can define variables in the source template using braces { }.

For example, consider the following example variables in a source template:

I would like to apply for the {role} at {company}.

You can define a variable on the source template page in the following ways:

1. Select the text you want to define as a variable and click the variable button.
2. Type the '{' and '}' characters around the text you want to convert to a variable.
3. Click the variable button, enter the variable name, and click finish. NOTE: There is a character limit on the variable name length in the template user interface. If you want to use a longer variable name, you must use 1 or 2 above to convert the text to a variable.

For example, the following shows the generated template for the preceding example variables:

I would like to apply for the role at company:

Text Automata displays the variable name in the generated template. To replace a variable, click the variable to edit it. Text Automata highlights variables that you select for editing.

For example, the following shows the variable role as highlighted for editing:

I would like to apply for the role at company.

Text Automata allows you to enter any text as a replacement for a variable in the generated template. You can also use another variable name as a replacement text for a variable. To copy and paste text into a variable, use the legacy generator linked at the bottom of the template page. You can create a new line by hitting enter. Hit the Esc key to finish editing variables. If you don't hit the Esc key after editing, you will not be able to edit other parts of the text.

For example, the following shows the replacement text for the role variable.

I would like to apply for the frontend developer role at company.

The following shows replacement text for both the role and company variables:

I would like to apply for the frontend developer role at Text Automata.

Here is another sentence you can generate with the same template:

I would like to apply for the keyboard designer position at Artemis Design Studios.

The autoapply setting assigns the same value to all variables with the same name. For example, consider the following source template with multiple instances of the company variable and one instance of the experience variable:

I was excited when I saw the opportunity at {company}. I believe that my experience in {experience} will allow me to thrive at {company}.

When you select company, Text Automata highlights all instances of company variable in the generated template:

I was excited when I saw the opportunity at company. I believe that my experience in experience will allow me to thrive at company.

You can now edit both variables simultaneously:

I was excited when I saw the opportunity at Text Automata. I believe that my experience in experience will allow me to thrive at Text Automata.

You can turn off autoapply by clicking the autoapply button. If autoapply is disabled, you can overwrite the text for the second company entry:

I was excited when I saw the opportunity at Text Automata. I believe that my experience in experience will allow me to thrive at Artemis Design Studios.

Nesting

You can nest variables within toggles. Here is an example of valid nesting:

[As an avid follower of the {industry}, I was excited about the opportunity at {company}.] [While I am unfamiliar with the {industry} I believe my experience in {experience} will allow me to thrive at {company}.]

If you exclude and hide a nested toggle, Text Automata hides the variables within the nested toggle.

Escape Character

You can use the escape character '\' (backslash) to enter text that would otherwise be evaluated. Here are some examples in a template:

1. arr[0]
2. arr\[0]
3. Here is a \\

Here is what the template evaluates to:

1. arr0
2. arr[0]
3. Here is a \

Legacy Generator

The original Text Automata generator provided a step-by-step interface to edit each segment individually. If you are concerned about forgetting variables or copying and pasting text, you may find the legacy generator more useful. The legacy generator supports hidden toggles and autoapply (in the form of autofill) as well. You can use the legacy generator by selecting the legacy button on the template page.

Preview

The preview button on the template page allows you to preview how your text will be semantically parsed. It displays the text without the parsed braces or brackets while including colors (like the examples on this page) to display a preview of the parsing.

Import and Download

You can import an existing template into the template page. The preferred file format is .txt for imports. You can download the template and the generated text as .txt files.
